\chapter{方法实现}\label{chap:float}

\section{统一状态表示与合法性判定}

六种方法采用同一套状态定义，避免因为表示方式不同而影响对比结论。长度为 $k$ 的元组
\begin{equation}
  s=(q_0,\ldots,q_{k-1})
\end{equation}
表示前 $k$ 列已放置皇后。尝试在下一列放入候选行 $r$ 时，合法性由下式判断：
\begin{equation}
  \text{safe}(s,r)=\bigwedge_{c=0}^{k-1}\left[(q_c\neq r)\land(|q_c-r|\neq |c-k|)\right].
\end{equation}
若条件成立，状态扩展为 $s'=s\oplus r$。

\section{M1：命令式穷举 + 逻辑筛选}

M1 先通过 \texttt{itertools.permutations} 枚举全部 $8!$ 个列-行映射，再借助 Kanren 的 \texttt{membero} 与对角线关系进行筛选。它的优点是实现直观、便于检查正确性；不足也很直接，即候选集在前期已经固定，几乎没有提前剪枝空间。

\section{M2：纯逻辑求解（\texttt{permuteq}）}

M2 以关系 \texttt{permuteq(rows, qs)} 描述“\texttt{qs} 是 \texttt{rows} 的一个排列”，再叠加对角线约束：
\begin{equation}
  \text{solution}(\mathbf{q})\Leftarrow \texttt{permuteq}(\mathbf{rows},\mathbf{q})\land\texttt{diag\_clear}(\mathbf{q}).
\end{equation}
这种写法更贴近逻辑编程范式，语义上简洁，但统一与回溯带来的解释器开销较大。

\section{M3：纯逻辑求解（自定义不等关系）}

\subsection{关系设计}

Kanren 0.2.3 中没有可直接复用的 \texttt{neq}，因此脚本补充了两类关系：
\begin{itemize}
  \item \texttt{neq\_rel(left, right)}：两项不相等；
  \item \texttt{diagonal\_neq\_rel(left, right, distance)}：两项差值绝对值不等于列距。
\end{itemize}

两类关系都在“按列赋值”的过程中即时触发，这一点是 M3 相比 M2 的关键差别：冲突更早暴露，后续搜索分支更少。

\subsection{关键代码}

\begin{lstlisting}[language=Python,caption={自定义不等关系实现},label=lst:custom-neq-rel]
def neq_rel(left, right):
    def goal(subst):
        left_value = reify(left, subst)
        right_value = reify(right, subst)
        if isvar(left_value) or isvar(right_value):
            return (subst,)
        if left_value == right_value:
            return ()
        return (subst,)

    return goal


def diagonal_neq_rel(left, right, distance: int):
    def goal(subst):
        left_value = reify(left, subst)
        right_value = reify(right, subst)
        if isvar(left_value) or isvar(right_value):
            return (subst,)
        if abs(left_value - right_value) == distance:
            return ()
        return (subst,)

    return goal
\end{lstlisting}

\section{M4 与 M5：DFS / BFS 图搜索}

两种图搜索都把“部分棋局”看作节点，把“在下一列放入合法皇后”看作扩展。差异仅在前沿容器：
\begin{itemize}
  \item DFS 使用栈，优先深入；
  \item BFS 使用队列，按层展开。
\end{itemize}

\begin{algorithm}
  \caption{统一搜索框架（DFS/BFS）}\label{algo:search-framework}
  \begin{algorithmic}[1]
    \Require 棋盘规模 $N$，容器类型 $C$（栈或队列）
    \Ensure 全部合法解集合 $S$
    \State 将空状态 $()$ 放入 $C$
    \State $S \gets \varnothing$
    \While{$C$ 非空}
    \State 取出状态 $state$
    \If{$|state| = N$}
    \State 将 $state$ 加入 $S$
    \Else
    \For{每个候选行 $r \in [0, N-1]$}
    \If{$\text{safe}(state, r)$}
    \State 将 $state \oplus r$ 放入 $C$
    \EndIf
    \EndFor
    \EndIf
    \EndWhile
    \State \Return $S$
  \end{algorithmic}
\end{algorithm}

\section{M6：位运算回溯}

M6 用位掩码维护列冲突和两条对角线冲突。对当前行而言，可放位置集合可写为
\begin{equation}
  \texttt{available}=\texttt{full\_mask}~\&~\sim(\texttt{col\_mask}|\texttt{main\_diag\_mask}|\texttt{anti\_diag\_mask}).
\end{equation}

每次取最低位可行位置并递归更新掩码，即可持续推进搜索。完整函数见附录脚本清单，正文不再重复展开。

\section{解集一致性验证}

六种方法都得到 92 个合法解；最小字典序解一致为
\begin{equation}
  (0,4,7,5,2,6,1,3),
\end{equation}
最大字典序解一致为
\begin{equation}
  (7,3,0,2,5,1,6,4).
\end{equation}
这说明六种实现虽然路径不同，但语义结果一致，可以在同一基线上比较性能。
