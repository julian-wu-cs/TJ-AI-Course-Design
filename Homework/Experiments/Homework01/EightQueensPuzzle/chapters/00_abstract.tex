\MakeAbstract{
    本报告以八皇后问题为实验对象，在同一套 Python 测试框架下比较六类求解策略：命令式全排列后逻辑筛选、纯逻辑 \texttt{permuteq}、补充不等关系后的纯逻辑求解、DFS、BFS 与位运算回溯。实验重点不只在“能否得到 92 个解”，还关注不同建模方式在约束触发时机和工程开销上的差异。

    针对 Kanren 0.2.3 中缺少内置不等约束这一现实问题，报告实现了可复用的 \texttt{neq\_rel} 与 \texttt{diagonal\_neq\_rel}，并将其接入按列增量约束流程。该做法把部分失败分支提前剪掉，使逻辑方法在不改变语义的前提下明显提速。

    在 Windows 11、Python 3.12.7、$8\times 8$ 棋盘、每种方法重复 3 次的设置下，六种实现均稳定返回 92 组解。平均耗时分别为：M1 1761.672 ms、M2 16507.190 ms、M3 2856.098 ms、DFS 6.003 ms、BFS 5.900 ms、位运算回溯 0.615 ms。结果说明：搜索树规模相近时，位级状态表示可显著压缩单节点处理成本；而在逻辑编程路径中，约束前置是降低运行时间的关键手段。整套代码与完整 JSON 结果已作为附录给出，便于复现实验与后续扩展到 $N$ 皇后。
